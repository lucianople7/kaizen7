# Memory Rules

## Preserve

- Decisions explicitly accepted by the user.
- Product architecture.
- Brand positioning.
- Supplier or market research with sources.
- SOPs and working processes.
- Constraints, discarded directions and reasons.

## Do Not Preserve As Fact

- Unverified brainstorms.
- Medical, legal, financial or regulatory claims without source.
- Temporary tool outputs.
- Draft copy that was not accepted.

## Before Big Recommendations

Check whether the vault already contains:

- Prior decision
- Similar research
- Existing SOP
- Contradiction with current proposal
- Open pending question

## Good Summary Shape

Use:

- What we know
- What was decided
- What is still uncertain
- What changed today
- Next action
