# Project Connectivity OS

## Decision

KAIZEN7, Flowmatik, THE FOCUX, Mr. Kaizen y la IP educativa deben mejorar como
proyectos individuales en repos separados, pero conectarse mediante contratos
pequenos.

No se mezclan responsabilidades. Se conectan salidas.

Este repo queda reservado para KAIZEN7: metaskills, memoria minima, rutas,
receipts, verificacion y contratos entre proyectos.

## Separacion

| Proyecto | Mejora propia | Exporta | Importa |
|---|---|---|---|
| KAIZEN7 | Kernel, metaskills, memoria, verificacion | rutas, receipts, writebacks, contratos | objetivos, evidencia, manifests |
| Flowmatik | Repo de fabricas de contenido y video | content packs, renders, storyboards | marca, dossiers, claims guard |
| THE FOCUX | Repo de marca premium, dossiers, confianza, futuro comercio | reglas de marca, evidencia, claims permitidos | research, assets, verificacion |
| Mr. Kaizen | Repo/canal de voz, guiones, comunidad, confianza | scripts, hooks, preguntas | dossiers, plantillas, guardrails |
| IP educativa | Repo o modulo de proyecto, no kernel | conceptos, retos, rutas | ciencia, voz, marca |

## Regla operativa

```text
Mejorar dentro del proyecto.
Exportar solo artefactos utiles.
Conectar por contrato.
Verificar antes de publicar.
Escribir memoria despues.
```

## Contratos principales

### Source to Signal

Fuente externa hacia KAIZEN7:

```text
source + summary + claims + risks + recommended_action
```

### Dossier to Content

THE FOCUX hacia Flowmatik:

```text
dossier_id + safe_angle + blocked_claims + story_hooks + visual_direction
```

### Content to Persona

Flowmatik hacia Kaizen:

```text
script + scene_context + cta + voice_constraints + claims_guard
```

### Artifact to Memory

Cualquier proyecto hacia KAIZEN7:

```text
artifact_path + decision + evidence + result + next_action
```

## Primera aplicacion

Cuando los repos externos existan, la cadena se conecta asi:

```text
THE FOCUX dossier
-> Flowmatik content pack
-> Kaizen script
-> Remotion render
-> KAIZEN7 verification
-> Obsidian writeback
```

KAIZEN7 solo debe guardar la ruta, el contrato, el receipt y la recomendacion
de memoria. La implementacion de dossier, video, web o contenido queda en el
repo del proyecto correspondiente.
