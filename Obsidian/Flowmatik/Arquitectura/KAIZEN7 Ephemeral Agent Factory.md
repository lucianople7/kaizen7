# KAIZEN7 Ephemeral Agent Factory

## Decision

KAIZEN7 will use short-lived specialist agents to create missing capabilities, adapters or mini-skills.

## Why

This magnifies Codex and other agents. KAIZEN7 gives direction, allowed files, forbidden actions, evidence requirements and learning capture instead of asking an agent to explore from zero.

## Loop

1. Need.
2. Forge packet.
3. Ephemeral agent brief.
4. Specialist builds adapter/capability/mini-skill.
5. Evidence gate.
6. Learning capture.
7. Register, revise, retire or archive pattern.

## Principle

The subagent is temporary. The learning is permanent.

## Guardrails

- One objective.
- Short lifetime.
- No install without approval.
- No paid/cloud/GPU default.
- No credentials.
- Tests and evidence required.
- Failed specialists leave a rejection reason, not permanent complexity.

## Product Thesis

KAIZEN7 converts each task into reusable capability learning so the next agent starts stronger.
