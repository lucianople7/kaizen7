# Intake - Graphify

**Sistema:** KAIZEN7
**Fecha:** 2026-06-25
**Veredicto:** `adapt_pattern`

## Que es

Graphify convierte proyectos en un grafo consultable con reporte, HTML y JSON. Permite preguntar al proyecto sin releer todos los archivos.

Salida principal:

```text
graphify-out/
  graph.html
  GRAPH_REPORT.md
  graph.json
```

## Patrones utiles

- Mapa consultable antes de lectura manual.
- Preguntas sugeridas para exploracion.
- Relaciones con nivel de confianza: extraidas, inferidas o ambiguas.
- Query-first para no gastar contexto releyendo.
- Posible MCP sobre `graph.json`.
- Memoria de resultados y reflexion sobre consultas utiles.

## Que se descarta

- Instalar Graphify ahora mismo.
- Hooks always-on.
- Enviar documentos, imagenes o PDFs a extraccion semantica sin decision explicita.
- Sustituir Obsidian, Product Genome o codigo local como fuente de verdad.

## Encaje en KAIZEN7

Graphify encaja como referencia para `K7 Memory Token Engine` y futura capa `K7 Context`.

No entra como core. Primero se absorbe la disciplina:

```text
intencion -> Obsidian -> indice/grafo -> lectura focalizada -> decision -> memoria
```

## Riesgos

- Puede indexar secretos si no se excluyen.
- Puede crear falsa confianza si el grafo esta viejo.
- Puede aumentar complejidad si se instala antes de necesitarlo.
- Algunos contenidos requieren llamadas a modelo externo.

## Adaptacion recomendada

Crear metaskill de memoria y tokens. Probar Graphify solo si se repiten exploraciones grandes o la web/KAIZEN7 crecen lo bastante para que `rg` + Obsidian no basten.
