# Less Steps, Less Tokens

KAIZEN7 should be easy, effective and cheap in context.

Core rule:

```text
Less steps. Fewer tokens. Better judgment.
```

## Super Effective

Super effective means KAIZEN7 does not do more to look smart.

It does less, better.

A feature, module, skill, adapter or screen enters only if it improves at least one operational metric:

- fewer steps,
- fewer tokens,
- lower risk,
- less repeated work,
- faster decision time,
- more clarity,
- stronger verification,
- more reusable learning.

Filter:

```text
If it does not improve an operational metric, it does not enter.
```

## Effectiveness Score

Before creating or keeping a piece, score it:

```text
Steps: - / 0 / +
Tokens: - / 0 / +
Risk: - / 0 / +
Clarity: - / 0 / +
Verification: - / 0 / +
Reuse: - / 0 / +
```

A piece is super effective when it has at least three `+` marks and no critical `-`.

If it does not pass:

- simplify it,
- merge it,
- postpone it,
- or remove it.

## Daily Mode

The daily loop is:

```text
Objective -> Action -> Verification + Memory
```

Do not open ten notes if one is enough.
Do not load ten skills if two solve the task.
Do not read the whole repository when an index and two files are enough.

## Short Commands

Move an objective forward:

```powershell
npm.cmd run k7 -- "objective"
```

Boost Codex or another agent:

```powershell
npm.cmd run k7:boost -- "objective"
```

Check readiness:

```powershell
npm.cmd run k7:ready
```

Run full verification:

```powershell
npm.cmd run k7:check
```

## Context Policy

KAIZEN7 should return the smallest useful package:

- what to read,
- what to avoid,
- what to do first,
- how to verify,
- where to write learning.

## Token Policy

1. Read indexes before long files.
2. Read canonical memory before searching the whole repo.
3. Use metadata before full content.
4. Load skills only when needed.
5. Prefer one verifiable action over a long list.
6. Write learning back so the next run needs less context.

## Compact Project Activation

Short version:

```text
Anchor -> Memory -> Action -> Verify -> Learn
```

Full version:

```text
Anchor -> Memory -> Map -> Modules -> Judgment -> Action -> Verification -> Evolution
```

Daily rule:

```text
If it does not reduce steps, risk or tokens, it does not enter.
```

Super effective rule:

```text
If it does not reduce friction or improve verifiable judgment, it does not enter.
```
