# Token Discipline

## Objetivo

Reducir tokens sin perder verdad.

## Regla

```text
Leer menos no significa saber menos.
Leer mejor significa saber lo justo para decidir.
```

## Escalera

1. Pregunta del usuario.
2. THE FOCUX Bible.
3. K7 Super IQ Engine.
4. K7 Memory Token Engine.
5. Nota del dominio afectado.
6. Busqueda con `rg`.
7. Archivo concreto.
8. Test o preview.
9. Memoria de cierre.

## Lo que se evita

- Dumps de carpetas.
- Listar caches.
- Leer dependencias.
- Abrir assets binarios salvo necesidad visual.
- Releer notas ya resumidas.
- Investigar internet si Obsidian ya contiene decision vigente y el dato no es temporal.

## Cuando si gastar tokens

Gastar contexto si protege:

- legalidad,
- claims,
- seguridad,
- dinero,
- publicacion externa,
- arquitectura central,
- decisiones irreversibles.

## Cierre

Cada sesion debe responder:

```text
Que lecturas evite?
Que memoria use?
Que aprendizaje reduce tokens la proxima vez?
```
