# Product Growth OS

## Decision

KAIZEN7 es la herramienta/agente. Debe funcionar con cualquier producto incorporado. THE FOCUX es el primer preset, no el limite del sistema.

## KAIZEN7 como producto

Definicion madre:

```text
KAIZEN7 es el cerebro operativo que ayuda a que cada proyecto llegue a su objetivo con menos pasos, menos tokens y mas criterio.
```

Categoria:

```text
Agentic Product Growth OS
```

Promesa:

```text
Objetivo -> memoria minima -> accion correcta -> verificacion -> aprendizaje guardado.
```

Regla:

KAIZEN7 no debe ser solo una coleccion de scripts. Debe comportarse como producto: claro, repetible, explicable, ampliable y vendible.

Documento canonico:

```text
Obsidian/Flowmatik/Producto/KAIZEN7 Product.md
```

## Separacion agente/proyecto

KAIZEN7 debe vivir como agente independiente. THE FOCUX debe vivir como proyecto independiente.

La relacion correcta es:

```text
KAIZEN7 -> gestiona -> THE FOCUX
```

No:

```text
KAIZEN7 = THE FOCUX
```

THE FOCUX puede ser el primer preset, caso real y proyecto principal, pero KAIZEN7 debe poder gestionar otros proyectos sin arrastrar la marca THE FOCUX.

## Capas operativas

Inspirado por el patron Alibaba Smart Assistant, KAIZEN7 organiza el crecimiento en cuatro capas:

```text
Setup -> Product -> Opportunity -> Risk
```

### Setup Agent

Reduce friccion inicial:

- intake,
- Product Genome,
- web/base,
- activos iniciales,
- canales,
- configuracion.

### Product Agent

Convierte producto/contenido en activos publicables:

- PDP,
- titulos,
- keywords,
- imagenes,
- videos,
- dossiers,
- comparativas,
- calendario.

### Opportunity Agent

No deja escapar demanda:

- leads,
- newsletter,
- partners,
- proveedores,
- afiliacion,
- follow-up,
- respuestas.

### Risk Agent

Protege confianza, compliance y margen:

- claims,
- regulacion,
- IP,
- seguridad,
- evidencia,
- disclaimers,
- guardrails.

## Regla

Cada producto se trata como una hipotesis de negocio validable antes de invertir fuerte en stock, desarrollo o anuncios.

## Adaptadores opcionales

KAIZEN7 puede integrar frameworks externos como adapters si mejoran una fase concreta sin sustituir el core.

Ejemplo:

- `Qwen Agent Adapter`: posible capa experimental para RAG, MCP, tool calling y agentes secundarios.

Regla:

```text
Adapter != Core
```

## Patrones externos

KAIZEN7 puede aprender de productos externos sin copiarlos.

- `Alibaba Smart Assistant Pattern`: referencia para organizar IA comercial en Setup, Product, Opportunity y Risk.

Regla:

```text
Pattern != Tool
```
