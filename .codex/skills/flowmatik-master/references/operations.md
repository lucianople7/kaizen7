# Operations

## SOP Format

Use:

- Purpose
- Inputs
- Owner
- Steps
- Quality checks
- Output
- Failure modes
- Update cadence

## Handoff Format

Use when a project becomes large or must continue later:

- Current objective
- Decisions made
- Open questions
- Files and assets
- Next actions
- Risks
- Suggested first prompt for next session

## Memory Rules

Preserve:

- Brand decisions
- Product architecture
- Customer definitions
- Reusable prompts
- SOPs
- Known constraints

Do not preserve:

- Random brainstorming that was not accepted
- Unverified claims as facts
- Tool outputs without business meaning

## Operating Cadence

Weekly:

- Review active opportunities
- Pick 1-3 priorities
- Create content pack
- Advance product or ecommerce asset
- Document what changed

Daily:

- One research action
- One production action
- One distribution action
