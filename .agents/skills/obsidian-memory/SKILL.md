# Obsidian Memory

## Overview

Usa el vault como memoria viva antes de proponer cambios importantes. Busca decisiones previas, resume contexto relevante y actualiza notas de forma conservadora.

## Vault Location

El vault principal del proyecto esta en:

```text
Obsidian/
```

Carpetas prioritarias:

1. `Obsidian/Flowmatik/`
2. `Obsidian/TheFocux/`
3. `Obsidian/Research/`
4. `Obsidian/Ideas/`
5. `Obsidian/Diario/`

## Search Protocol

Antes de responder en tareas de negocio, producto, contenido o ecommerce:

1. Buscar notas relacionadas.
2. Leer solo las notas necesarias.
3. Separar:
   - decisiones confirmadas
   - hipotesis
   - ideas descartadas
   - pendientes
4. Usar decisiones previas antes de proponer cambios.
5. Si se actualizan notas, preservar el contexto existente y anadir secciones fechadas.

Buscar sinonimos utiles:

- `THE FOCUX`, `TheFocux`, `FOCUX`
- `NeuroCity`, `nootropicos`, `peptidos`
- `Flowmatik`, `KAIZEN7`, `agentes`, `SOP`, `roadmap`
- `proveedores`, `claims`, `regulacion`, `contenido`

## Note Writing Rules

- Usar Markdown simple.
- Mantener titulos claros.
- No mezclar diario, research y decisiones finales en una sola nota.
- Marcar incertidumbre: `Pendiente de validar`, `Hipotesis`, `Decision`.
- No convertir claims medicos, financieros o legales en hechos sin fuente.

## Update Format

Cuando actualices una nota, preferir:

```markdown
## Actualizacion YYYY-MM-DD

### Decision

...

### Contexto

...

### Siguiente accion

...
```
