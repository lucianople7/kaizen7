
API:

```http
POST /api/k7/cockpit
```

Decision: KAIZEN7 debe empezar preguntando el objetivo, pedir contexto minimo y construir solo la toolchain necesaria. No megatool, no lectura masiva, no tokens quemados sin motivo.

## Orientacion de producto

KAIZEN7 queda orientado como `goal-to-execution layer`:

```text
Objective -> Minimal Context -> Memory -> Toolchain -> Execution -> Verification -> Learning
```

Tesis:

```text
Less context. Fewer steps. Verified action.
```

Uso principal: ayudar a asistentes de codigo, agentes y creadores de proyectos a llegar a un objetivo concreto con menos tokens, menos pasos y mas evidencia.

## Metaskill Boot

Se anadio `metaskillBoot` al Activation Cockpit.

Decision: el arranque de KAIZEN7 debe entregar tambien las metaskills necesarias para el objetivo, con orden de activacion e instrucciones para el editor/agente. Esto permite que Codex, OpenHands u otro editor empiece con nuestras metaskills sin cargar todo el sistema.

Regla:

```text
Objetivo -> contexto minimo -> metaskills necesarias -> toolchain minima -> accion -> verificacion -> memoria
```
