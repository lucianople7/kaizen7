# KAIZEN7 Mission Control

## Purpose

KAIZEN7 Mission Control is the first "JARVIS-like" entrypoint for the ecosystem.

It is not a new agent. It is the command layer that turns a human objective into the objects an agent needs.

```text
Human decides.
KAIZEN7 coordinates.
Agents execute.
Projects evolve.
```

## Route

```text
Human objective
-> Growth Gate
-> K7 Mission
-> Capability Resolver
-> Mission Brief
-> Agent Execution
-> Mission Outcome Receipt
-> PR/Review
-> Memory Recommendation
```

## What It Produces

- growth gate
- normalized mission
- capability resolution
- Mission Brief
- Mission Outcome Receipt template
- next action
- commands for brief, outcome and readiness

## What It Does Not Do

- It does not execute code.
- It does not publish.
- It does not deploy.
- It does not spend money.
- It does not write memory automatically.
- It does not replace Codex, ChatGPT, Claude, Qwen or other agents.

## Growth Gate

Before a mission reaches an agent, Mission Control checks:

- growth lane,
- valid output,
- executable or rewrite required.

Valid growth lanes:

- THE FOCUX Growth
- Flowmatik Production
- Kaizen Audience
- KAIZEN7 Execution Quality

Valid outputs:

- published content,
- reusable asset,
- validated decision,
- lead or audience action,
- product or sale,
- time or friction reduction.

Mission Control prepares the work. Agents do the work.
