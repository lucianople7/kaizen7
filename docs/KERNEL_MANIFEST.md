# KAIZEN7 Small Kernel Manifest

KAIZEN7 should stay small at the center.

It only needs to answer four questions:

1. What does the user want to achieve?
2. Which capability solves that objective?
3. Which provider is best available now?
4. Which evidence proves the result is good enough?

Everything else is replaceable: Codex, ChatGPT, OpenHands, CLI-Anything, FFmpeg, OpenCut, Remotion, Kdenlive, Shotcut, ComfyUI or any future provider.

## Minimal capabilities

- research: find and compress trustworthy inputs.
- create: turn ideas into usable artifacts.
- compare: contrast options, risks and claims.
- render: prepare output files without publishing.
- publish: plan external release with approvals.
- remember: save reusable learning, not raw noise.
- verify: close only when evidence exists.

## First validation flow

The first validation flow is a Mr. Kaizen short content package:

`research -> create -> compare -> render -> remember -> verify`

It does not publish. It produces a reviewable script, storyboard, claims guard, artifact manifest and Obsidian memory note.
